4. Cross-screen copy rules
These rules should be enforced across the interface.
Rule 1
Never use “Resolved” copy while:
communication is still required and pending
a meaningful dependency remains open
the system is still in Action In Progress
Rule 2
Never use vague language like:
“Everything is okay”
“Issue handled”
“Done” without operational truth behind it.
Rule 3
When risk increases, copy becomes:
shorter
more directive
less interpretive
Rule 4
When the system lacks confidence, copy must say so clearly:
Recommendation confidence is low
No safe recommendation is available
Manual recovery is required
Rule 5
When a case is partially resolved, the screen must name the remaining open dependency explicitly.

6. Final recommendation
The minimum believable MVP surface for this product is not just:
dashboard,
detail,
decision,
confirmation,
progress,
resolution.
It also requires:
communication review,
recovery,
lock/conflict visibility,
and recalculation interruption.
Without those, the product looks intelligent but behaves incompletely.
